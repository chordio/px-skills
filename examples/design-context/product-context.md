# Product Context

## What We're Building

**Product Name:** TaskFlow
**Type:** Project management SaaS
**Core Value:** Simple, visual task management for small teams

## Target Users

### Primary Persona: Sarah (Team Lead)

**Background:**
- Manages a team of 5-8 people
- Needs visibility into team progress
- Juggles multiple projects simultaneously

**Goals:**
- See team workload at a glance
- Quickly assign and reassign tasks
- Track project milestones

**Pain Points:**
- Complex tools slow her down
- Notifications overwhelm her
- Hard to get team buy-in on tools

**Expectations:**
- Fast, responsive interface
- Clear visual hierarchy
- Minimal training required

### Secondary Persona: Alex (Individual Contributor)

**Background:**
- Developer/designer on the team
- Focuses on executing tasks
- Wants minimal overhead

**Goals:**
- Know what to work on next
- Update task status quickly
- Stay focused, avoid distractions

**Pain Points:**
- Too many status update requests
- Unclear task priorities
- Context switching between tools

## Industry Context

**Market:** Project management / productivity tools
**Competitors:** Asana, Trello, Linear, Monday
**Differentiator:** Simplicity over features

## Key Product Areas

1. **Dashboard** - Overview of all projects and tasks
2. **Project View** - Kanban board for single project
3. **Task Detail** - Full task information and activity
4. **Team View** - Workload across team members
5. **Settings** - User and workspace configuration

## Design Principles

1. **Speed First** - Every interaction should feel instant
2. **Progressive Disclosure** - Show complexity only when needed
3. **Visual Clarity** - Clear hierarchy, obvious actions
4. **Forgiving** - Easy to undo, hard to break things
